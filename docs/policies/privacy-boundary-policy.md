# Privacy Boundary Policy

## Rule

Atlas must distinguish clearly between:
- public-safe content
- low-risk business configuration data
- protected or sensitive customer data

## Public-safe content
May exist in Wix and GitHub when approved.
Examples:
- public package descriptions
- approved claims
- public FAQs
- generic workflows

## Protected or sensitive content
Must not be stored in public shells or governance repos.
Examples:
- customer-sensitive documents
- protected package outputs
- secrets and credentials
- live entitlement state
- protected delivery records

## Enforcement principle

If a workflow requires protected execution, revocable access, signed delivery, or auditable handling, it belongs in the secure runtime.

# Protected Delivery Contract

## Purpose
Define the minimum requirements for delivering protected or managed Atlas outputs.

## Requirements
- delivery must be revocable where feasible
- access must be tied to an authenticated or approved recipient
- delivery events must be logged
- protected artifacts must not be exposed directly from Wix or GitHub Pages
- sensitive runtime state must remain in the secure runtime plane

## Minimum delivery states
- pending
- approved
- delivered
- revoked
- failed
